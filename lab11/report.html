Objective

The objective of Lab 11 was to incorporate Business Intelligence (BI) into the Pine Valley Furniture Company (PVFC) system. This involved implementing a Recommendation System based on Market Basket Analysis using SQL Self-Joins and a dedicated Web API for serving suggestions.

Key Functionalities Implemented

Feature Implementation Details Technical Logic
Market Basket Analysis Identifies products frequently bought together. SQL SELF JOIN on Order_line_t.
Recommendation Web API A separate Blazor Web App (ASP.NET Core) serving JSON. Decoupled logic running on a separate CPU (simulated via https://localhost:7087).
Real-time Suggestions "Suggest" button in Search.aspx fetches data via AJAX. JavaScript fetch() API calling the recommendation endpoint.
Reordering System Personalized suggestions in Order.aspx. SELECT DISTINCT from past orders for the logged-in user.
Inventory Forecasting Manager dashboard for demand prediction. Frequency analysis of product pairings to predict secondary demand.

SQL Self-Join Logic (Market Basket)

To find items bought with a specific product, we join the Order_line_t table with itself on the same Order_Id:


SELECT TOP 3 p2.Product_Description, COUNT(*) as Frequency
FROM Order_line_t l1
JOIN Order_line_t l2 ON l1.Order_Id = l2.Order_Id AND l1.Product_Id <> l2.Product_Id
JOIN PRODUCT_t p2 ON l2.Product_Id = p2.Product_Id
WHERE l1.Product_Id = @CurrentID
GROUP BY p2.Product_Id, p2.Product_Description
ORDER BY Frequency DESC;

System Architecture

  • Backend: ASP.NET Core Controller (RecommendationController.cs) in a separate Blazor project.
  • Database: SQL Server with associations established via order history.
  • Frontend: AJAX-enabled ASP.NET Web Forms for a dynamic user experience without full page reloads.
