Topic: Recommendation System and Market Basket Analysis (BI)

Problem Statement

Incorporate business intelligence in your PVFC order processing system by adding a recommendation system based on Market Basket Analysis using Self Joins.

Requirements:
  1. Market Basket Analysis: Use the concept of Self Joins to identify products that are frequently bought together. "If a user buys product A, he will likely buy product B."
  2. Web API: Implement a Web API (using Generic Handlers or ASP.NET Web API) to serve recommendations. The API should accept a Product ID and return suggested items.
  3. Recommendation Messages: Display "Customers who bought this also bought..." during Product Selection and Order Placement.
  4. Reordering Suggestions: Provide personalized reordering suggestions for customers based on their purchase history.
  5. Inventory Forecasting: Develop a manager/employee dashboard to forecast demand for secondary items when a primary item's sales increase.
Technical Constraints:
  • Use Self Join in SQL for association rule mining.
  • Separate the API logic from the main server processing (simulate separate CPU via Web API).
  • Use AJAX to fetch recommendations from the Web API in real-time.
